registry-pattern

PythonDesign Patterns

registry-pattern provides name-based registries for classes and functions plus a recursive factory that builds object graphs from JSON-friendly envelopes. A TypeRegistry or FunctionalRegistry collects artifacts by name via a register decorator; build() then consumes a {type, repo, data, meta} envelope, validates the data against the target's signature, and recurses on nested envelopes to assemble whole pipelines from config. serialize() is the inverse, turning a live object graph back into the same envelope for round-tripping.

Registries are tree-shaped and pass post_init / serialize_meta / pre_call / post_call hooks down by inheritance, so a sub-registry can add validation or emit provenance into the meta dict, and optional meters and reporters layer observability on top. The core depends only on Pydantic and typing-extensions. Published on PyPI as registry-pattern.
