torchestrator

PythonPyTorch

torchestrator is a Dockerfile-like declarative experiment runner for PyTorch. One experiment is a "torchfile": a YAML-ish config that names the model, dataset, optimizer, loss, metrics and training loop, with FROM inheritance between torchfiles. The runner builds and runs it, and on success pickles the live objects (model, dataset, optimizer, scheduler) into a content-addressed hub at ~/.cache/torchestrator/, so a child torchfile that FROMs a parent inherits the trained objects on cascade load, with no state_dict surgery or manual checkpoint paths.

By design it is not an experiment tracker, hyperparameter sweeper or job scheduler: it wires to W&B / MLflow / TensorBoard for telemetry, pairs with Optuna / Ray Tune for search, and delegates execution to srun / apptainer / docker. It is the runner the paper-reproductions monorepo builds on. (v0.7.0, alpha; repository currently private.)
