observer-pattern

PythonDesign Patterns

observer-pattern (published as eventforge) is a message-driven execution and event library. Its Observable / Eventful / Dispatcher core lets any object emit events; on top sit a topic-keyed MessageQueue (pub/sub with Pydantic-validated messages), a competing-consumer WorkQueue with ack / nack / dead-letter, and an Executor that runs work sequentially, across threads, or across processes. A single @task decorator makes a function callable both directly and through the queue, with the same success / failure and observer hooks on either path.

Queue-RPC adds request / reply on top of the queue, and a RemoteQueue bridges queues across nodes over TCP for distributed pipelines; built-in Meter and Reporter observers profile timing, metrics, memory and CPU without touching task code. It is the dispatch and execution half that pairs with registry-pattern's serialization.
